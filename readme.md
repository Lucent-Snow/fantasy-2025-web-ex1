# 练习1：Fantasy 动漫社官网页面制作

这是 Fantasy 动漫社技术组第一次综合练习。本任务将带你完整体验一次简单的前后端协作开发流程，包括网页布局、动态交互、后端接口设计以及 Git 协作。

请确保你已经了解基本的 HTML、CSS、JavaScript 使用方法。

---

## 目标内容

- 熟悉前端页面结构与样式编写
- 实现基本的前后端数据交互
- 练习 Git 分支开发与 Pull Request 流程

---

## 项目目录要求

本次作业需要使用以下目录结构：

```
project/
│
├─ src/ # 前端页面源代码
│   ├─ index.html
│   ├─ style.css (可选)
│   └─ main.js / main.ts / main.py ...
│
├─ assets/ # 前端资源（图片等）
│
└─ backend/
    ├─ src/ # 后端源代码
    │   └─ server.js / server.ts / main.py ...(任选)
    └─ assets/ # 后端需要使用的资源（如有）
```

> 注：一般不会将大量资源直接放进 git 仓库，这里仅用于练习。

---

## 前端任务

请设计 **Fantasy 动漫社官网主页**，包含：

1. 首页导航，展示：
    - 网页小游戏入口
    - 纳新报名入口
    - 作品展示页面入口

2. 在首页添加一个 **图片展示栏**：
    - 页面加载时随机展示一张图片
    - 图片可定时自动切换（可使用 `setInterval`）
    - 同时提供“下一张”按钮手动切换

3. 所有前端代码放在 `src/` 文件夹内，资源文件放在 `assets/` 文件夹内。

---

##  后端任务

实现一个简单的 Web 服务用于处理数据交互。

推荐语言：Node.js（也可以使用 Python / Go 等）

后端要求：

1. 将前端网页通过 HTTP 服务在本地访问，端口可为 `11451`（或你自行设定）
2. 访问根路径 `/` 时返回 `src/index.html`
3. 实现接口：

```
POST /api/groupSize
参数：year （例如 2024 或 2025）
返回该年份群人数（JSON 格式）
```

测试数据（截至 11 月 10 日）：

| 年份   | 群人数 |
|------|-----|
| 2025 | 737 |
| 2024 | 834 |

示例返回：

```json
{ "size": 737 }
```
